# code-trace UI 布局优化设计规格说明

**日期**: 2026-06-08
**状态**: 已批准
**范围**: 优化 Tool Window 的三栏布局、节点操作组织、行内常用操作、链接源可视化与操作反馈

## 1. 背景

当前 `code-trace` Tool Window 已经支持 trace 文件切换、节点表格、节点编辑、节点移动、链接、备注保存、展开/折叠和源码跳转。现有 UI 的主要问题是视觉层次不够清晰：文件列表、节点操作按钮、节点表格、备注区和链接状态的权重接近，用户不容易判断主要工作区和当前操作上下文。

本次优化聚焦在已确认的高优先级范围：

- 重构为三栏结构：左侧 trace 文件列表、中间节点主工作区、右侧备注侧栏。
- 将节点操作按钮按功能分组。
- 将 `Edit Node`、`Delete Node`、`Move Up`、`Move Down` 移入表格行内操作列。
- 增加链接源可视化、按钮启用态和操作反馈。

## 2. 非目标

本次不实现以下内容：

- 左侧节点树或 trace 文件内节点层级树。
- 右键上下文菜单。
- 表格排序。
- 新增批量删除、批量移动等多选批量操作。
- trace JSON schema 变更。
- 独立 toast 系统或复杂动画反馈。

现有多选和拖拽行为如果已经存在，应继续保留；本次不扩大其功能范围。

## 3. 总体布局

Tool Window 采用三栏工作区，不改变数据模型。

### 3.1 左侧 trace 文件侧栏

左侧继续使用现有 `TraceFileListPanel`，负责 trace 文件选择和文件级操作。

保留现有能力：

- 创建 trace 文件。
- 重命名 trace 文件。
- 复制 trace 文件。
- 删除 trace 文件。
- 刷新 trace 文件列表。

左侧不展示节点树。节点层级仍由中间节点表格中的编号、缩进和展开/折叠状态表达。

### 3.2 中间节点主工作区

中间区域获得最高视觉权重，包含：

- 顶部节点工具栏。
- 节点表格。
- 底部状态栏。

表格是主工作区的核心，应占据中间区域主要高度和宽度。工具栏和状态栏作为辅助层级展示。

### 3.3 右侧备注侧栏

右侧合并备注能力，包含：

- Trace Note。
- Node Note。

两个备注区垂直排列。Trace Note 位于上方，Node Note 位于下方。`Save Trace Note` 和 `Save Node Note` 使用次级视觉样式，避免和节点操作争夺注意力。

### 3.4 顶部全局栏

顶层全局栏只保留全局动作：

- `Refresh`
- `Toggle Files`

节点编辑、链接和视图控制不再放在顶层全局栏中。

## 4. 节点操作组织

### 4.1 工具栏分组

中间主区顶部工具栏只保留跨节点上下文操作，并用视觉分隔组织为两组。

链接组：

- `Set as Source`
- `Link To Here`
- `Unlink`
- `Go to Linked Node`

视图组：

- `Expand All`
- `Collapse All`

按钮文本统一使用 title case。原 `Go To Linked` 改为 `Go to Linked Node`，明确目标是节点而不是抽象链接。

### 4.2 行内常用操作

以下常用节点操作从工具栏移入表格最右侧 `操作` 列：

- `Edit Node`
- `Delete Node`
- `Move Up`
- `Move Down`

行内操作使用图标按钮和 Tooltip，不使用文字按钮，避免压缩节点名称列。按钮点击时作用于所在行节点，不依赖用户再次确认当前选中行。

`Delete Node` 使用危险操作表现，并在执行前弹出确认对话框。

## 5. 节点表格设计

表格列调整为：

1. `编号`
2. `节点名称`
3. `链接关系`
4. `操作`

列宽原则：

- `编号` 列保留紧凑宽度，继续承载展开/折叠和层级显示。
- `节点名称` 列继续作为弹性主列，承担主要阅读内容。
- `链接关系` 列保留中等宽度。
- `操作` 列固定较窄宽度，只容纳四个图标按钮。

实现时必须保留最近已修复的列宽行为：表格重建、拖拽和移动操作后，用户调整过的列宽不应丢失，最后一列填充视口的逻辑不应回退。

## 6. 链接源可视化

链接流程保持两步：

1. 用户选中源节点并点击 `Set as Source`。
2. 用户选中目标节点并点击 `Link To Here`。

状态规则：

- 未设置链接源时，`Link To Here` 禁用。
- 设置链接源后，`Link To Here` 启用。
- 当前链接源节点行显示轻量高亮，例如左侧色带或浅蓝色背景。
- 如果链接源节点被删除、当前文件切换或刷新后不存在，应清除链接源状态。

底部状态栏文案：

- 未设置：`链接源：未设置，请先选中节点并点击 Set as Source`
- 已设置：`链接源：#<编号> <节点名称>`

节点名称过长时，状态栏可以截断显示，但 Tooltip 或后续选择状态应仍能识别完整节点。

## 7. 操作反馈

本次使用中间主区底部状态栏提供操作反馈，不新增独立 toast 系统。

状态栏负责展示：

- 当前链接源状态。
- 最近一次操作结果。

需要反馈的操作包括：

- 移动节点成功。
- 设置链接源成功。
- 链接成功。
- 取消链接成功。
- 删除节点成功。
- 保存 Trace Note 成功。
- 保存 Node Note 成功。

反馈应短暂展示后回到链接源状态，或在下一次选择/操作时自然刷新。

错误或无效操作应显示可理解的状态文案，例如未选择节点、目标节点无链接、链接端点不存在等。

## 8. 组件边界

### 8.1 `TraceEditorPanel`

职责：

- 组织编辑区布局。
- 创建中间主区工具栏。
- 创建右侧备注侧栏。
- 配置节点表格基础外观。
- 提供按钮、表格、文本区、状态栏组件访问器。

`TraceEditorPanel` 不直接修改 trace 数据。

### 8.2 `CodeTracePanel`

职责：

- 绑定 UI 事件。
- 协调 `CodeTraceController` 的节点编辑、删除、移动、链接、跳转、保存备注等操作。
- 维护链接源状态。
- 维护选中行同步。
- 更新状态栏反馈。
- 管理删除确认。

### 8.3 表格操作列

新增表格操作列相关渲染和编辑能力，用于在每行显示图标按钮。

职责：

- 将 `Edit Node`、`Delete Node`、`Move Up`、`Move Down` 显示为图标按钮。
- 根据行节点触发现有操作逻辑。
- 提供 Tooltip。
- 避免破坏表格选择、拖拽和列宽逻辑。

新增类应放在 `com.zimaai.codetrace.toolwindow` 包内。

## 9. 数据流

### 9.1 行内操作

```
用户点击某行操作图标
    ↓
表格操作列解析所在行的 TraceNode
    ↓
CodeTracePanel 调用现有节点操作逻辑
    ↓
CodeTraceController 修改 TraceDocument
    ↓
保存并刷新表格
    ↓
状态栏显示结果
```

### 9.2 链接流程

```
用户选择源节点
    ↓
点击 Set as Source
    ↓
CodeTracePanel 保存 linkSourceNodeId
    ↓
表格高亮源节点，状态栏显示源节点
    ↓
用户选择目标节点
    ↓
点击 Link To Here
    ↓
CodeTraceController 创建链接
    ↓
刷新链接关系列和状态栏
```

### 9.3 备注保存

```
用户编辑 Trace Note 或 Node Note
    ↓
点击对应 Save 按钮
    ↓
CodeTracePanel 调用保存逻辑
    ↓
CodeTraceController 保存文档
    ↓
状态栏显示保存成功
```

## 10. 错误处理

- 删除节点前必须显示确认对话框。
- 删除当前链接源节点后，应清除链接源状态并禁用 `Link To Here`。
- 当前选中节点没有有效链接时，`Go to Linked Node` 应禁用。
- 链接端点缺失时，导航按钮应禁用，并避免抛出 UI 异常。
- 切换 trace 文件后，应重建表格并清除不再有效的链接源状态。
- 刷新当前文件后，如果首选选中节点仍存在，应保留选中；如果不存在，应清理相关状态。

## 11. 测试范围

需要新增或更新 `toolwindow` 相关测试。

重点覆盖：

- 顶部全局栏仅包含 `Refresh` 和 `Toggle Files`。
- 中间节点工具栏包含链接组和视图组按钮。
- `Edit Node`、`Delete Node`、`Move Up`、`Move Down` 不再作为主工具栏按钮出现。
- 表格包含 `操作` 列。
- 行内操作列能触发编辑、删除、上移、下移逻辑。
- 未设置链接源时，`Link To Here` 禁用。
- 设置链接源后，`Link To Here` 启用，状态栏显示源节点，源节点行有高亮。
- 删除链接源节点后，链接源状态被清除。
- `Delete Node` 走确认路径。
- 保存 Trace Note 和 Node Note 后，状态栏显示反馈。
- 表格重建和移动后，列宽持久逻辑不回退。

## 12. 验收标准

- Tool Window 呈现左侧文件侧栏、中间节点主工作区、右侧备注侧栏。
- 节点表格成为视觉主区域，备注区和文件区不抢主视觉权重。
- 节点操作按钮按链接、视图、行内常用操作分组。
- 常用行操作能直接作用于对应行节点。
- 链接流程通过按钮启用态、源节点高亮和状态栏文案清楚表达。
- 删除操作有确认，危险行为不易误触。
- 现有 trace 文件读写、节点链接、节点移动、节点删除、备注保存、展开/折叠和源码跳转功能继续工作。
- 本次实现不改变 trace JSON schema。
